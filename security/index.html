Certifications & Compliance

Third-party verified security standards

SOC 2 Type II

We undergo annual SOC 2 Type II audits conducted by an independent CPA firm. Our controls covering security, availability, and confidentiality are verified every year.

GDPR Compliant

We comply with the General Data Protection Regulation. Data Processing Agreements are available for EU customers. Our DPO can be reached at dpo@clayflow.com.

CCPA Ready

We support California Consumer Privacy Act requirements, including data access and deletion requests. All CCPA rights are fully automated through the platform.

ISO 27001

Our Information Security Management System is certified under ISO/IEC 27001:2022, covering all infrastructure, operations, and development processes.

Encryption Standards

Your data is encrypted at all times

Encryption at Rest

All data is encrypted using AES-256 with keys managed through a dedicated key management service. Database volumes, backups, and file storage are all encrypted. Encryption keys are rotated automatically every 90 days.

Encryption in Transit

All traffic between clients and our servers is encrypted with TLS 1.3. We enforce HSTS and maintain an A+ SSL rating. Internal service-to-service communication is also encrypted using mutual TLS.

End-to-End Encryption

Enterprise customers can enable end-to-end encryption for sensitive data fields. Encryption keys can be customer-managed through AWS KMS or Azure Key Vault integration.

Key Rotation

Encryption keys are automatically rotated every 90 days. Previous keys are retired within 30 days. We support Bring Your Own Key (BYOK) for Enterprise plans.

Infrastructure Security

Secure by design, built on AWS

Network Security

All infrastructure runs in isolated VPCs with no public access to database or internal services. WAF, DDoS protection, and intrusion detection are active at all layers.

Access Control

Least-privilege access with mandatory MFA for all employees. Access reviews are conducted quarterly. Production access requires approval and is logged and audited.

Monitoring & Detection

24/7 security monitoring with real-time alerting. SIEM integration, anomaly detection, and automated threat response. Our SOC team responds to incidents within 15 minutes.

Backup & Recovery

Automated daily backups with point-in-time recovery. Cross-region replication ensures disaster recovery capability with an RTO of 4 hours and RPO of 15 minutes.

Vulnerability Management

Continuous vulnerability scanning across all infrastructure. Critical vulnerabilities are patched within 24 hours. Monthly penetration tests by independent third-party firms.

Audit Logging

Immutable audit logs capture all access and changes to the system. Logs are retained for 1 year and can be exported to your SIEM via our integration.

Application Security

Security built into our development lifecycle

Secure Development Lifecycle

All code is reviewed using automated SAST, SCA, and secret scanning tools before deployment. We follow OWASP Top 10 guidelines and require signed commits with GPG keys.

Authentication & Authorization

We support SAML 2.0, SSO via Google Workspace, Microsoft Entra ID, Okta, and OneLogin. Role-Based Access Control (RBAC) allows granular permission management. MFA is enforced for all accounts.

API Security

All API requests require authentication via API keys or OAuth 2.0. Rate limiting, request validation, and anomaly detection are enforced at the API gateway.

Bug Bounty & Disclosure Program

Help us keep Clayflow secure

We believe in responsible disclosure. If you discover a security vulnerability, please report it to us immediately. We offer rewards of up to $50,000 for qualifying vulnerabilities.

How to Report

Email security@clayflow.com with details of the vulnerability. Please include steps to reproduce, affected versions, and your contact information. We will acknowledge receipt within 24 hours.

Our Commitment

We will investigate all reports promptly and keep you informed of progress. We commit to no legal action against researchers who follow responsible disclosure practices.

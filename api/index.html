API Reference

Build on Clayflow

Our RESTful API gives you programmatic access to the full Clayflow platform. Authenticate with API keys, receive webhook events, and integrate deeply with your stack.

Explore Endpoints Authentication

API Overview

https://api.clayflow.io

Base URL for all API requests

v1

Current API version (latest stable)

JSON

Request and response format

// Example request
curl -X GET "https://api.clayflow.io/v1/projects" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

Core Endpoints

GET /v1/projects
List

Retrieve a paginated list of all projects in your workspace. Supports sorting and filtering by status, date, and tags.

Show example response
{
  "data": [
    { "id": "proj_abc123", "name": "Q4 Campaign", "status": "active" },
    { "id": "proj_def456", "name": "API Integration", "status": "draft" }
  ],
  "pagination": { "page": 1, "total": 42 }
}
POST /v1/projects
Create

Create a new project in your workspace. Accepts a name, description, and optional configuration parameters.

Show example request
curl -X POST "https://api.clayflow.io/v1/projects" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "New Project", "description": "Launch campaign", "template": "blank"}'
PUT /v1/projects/:id
Update

Update an existing project's attributes. Only include fields that you want to modify.

Show example
curl -X PUT "https://api.clayflow.io/v1/projects/proj_abc123" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"name": "Updated Name", "status": "archived"}'
DELETE /v1/projects/:id
Delete

Permanently delete a project and all associated data. This action cannot be undone.

Show example
curl -X DELETE "https://api.clayflow.io/v1/projects/proj_abc123" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  // Returns 204 No Content

Authentication

API Keys

All API requests require authentication via a Bearer token in the Authorization header. Generate and manage your API keys from the dashboard settings.

// Recommended: use environment variables
Authorization: Bearer ${CLAYFLOW_API_KEY}

Best Practices

  • Store API keys in environment variables, never in code.
  • Rotate keys every 90 days via the dashboard.
  • Use separate keys for development and production.
  • Revoke compromised keys immediately.

Rate Limits

Requests per minute (Pro)

Custom rate (Enterprise)

Rate limit headers: Every response includes X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset. Exceeding the limit returns a 429 status code.
